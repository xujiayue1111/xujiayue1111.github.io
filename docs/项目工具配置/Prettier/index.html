配置 Prettier

eslint 针对的是 javascript，它是一个检测工具，包含 js 语法以及少部分格式问题，在 eslint 看来，语法对了就能保证代码正常运行，格式问题属于其次；

而 prettier 属于格式化工具，它看不惯格式不统一，所以它就把 eslint 没干好的事接着干，另外，prettier 支持

包含 js 在内的多种语言。

总结起来，eslint 和 prettier 这俩兄弟一个保证 js 代码质量，一个保证代码美观。

安装依赖包​

pnpm install -D eslint-plugin-prettier prettier eslint-config-prettier

配置两个文件​

使用.prettierignore文件忽略不检查格式的文件​

/dist/ /html/ .local /node_modules/* **/.svg */.sh /public/*

使用.prettierrc.json文件配置规则。​

参考官方文件去书写规则

{
  要求字符串全部为单引号
  "singleQuote": true,  
  语句后不加分号
  "semi": false,
  "bracketSpacing": true,
  "htmlWhitespaceSensitivity": "ignore",
  "endOfLine": "auto",
  "trailingComma": "all",
  缩进占用两个文件
  "tabWidth": 2
}

安装完依赖之后同样使用pnpm run lint去检查格式错误，使用pnpm run fix去修改格式错误。​
