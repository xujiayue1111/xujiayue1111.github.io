axios二次封装

在开发项目的时候避免不了与后端进行交互,因此我们需要使用axios插件实现发送网络请求。在开发项目的时候

我们经常会把axios进行二次封装。

目的:

1.使用请求拦截器，可以在请求拦截器中处理一些业务(开始进度条、请求头携带公共参数)

2.使用响应拦截器，可以在响应拦截器中处理一些业务(进度条结束、简化服务器返回的数据、处理http网络错误)

utils文件夹可以用来放项目中常用的工具性文件，在src下创建这个文件夹。在utils里创建request.ts并在内部写入下面代码。

二次封装axios​

// 这里的文件对axios进行二次封装：使用请求和响应拦截器
// 首先引入axios
import axios from 'axios'
import { ElMessage } from 'element-plus'
// 利用axios对象的create方法创建axios实例（可以进行其他的配置，比如说基础路径或者超时时间）
// 这个对象一旦执行会返回一个axios实例对象，这里使用request进行接收。
// import axios from 'axios'以及let request = axios.create() 里面axios和request是同样的东西都是axios
// 只不过使用create创建的axios可以进行一些其他的配置并且还带着请求和响应拦截器
let request = axios.create({
    // 配置发送请求的基础路径
    // 这个基础路径是我的开发环境里面的变量
    // 这里因为接口的开头都是/api，所以开发环境变量里的VITE_APP_BASE_API也变成了/api
    baseURL: import.meta.env.VITE_APP_BASE_API, //基础路径上会自动的携带/api
    timeout: 5000 //超时时间设置成5秒
})
// 给axios实例request添加请求和响应拦截器
// 请求拦截器
// 在发送请求之前就会触发这里的请求拦截器，请求拦截器被触发时，这里的回调函数会传入配置对象
request.interceptors.request.use((config) => {
    // config配置对象，里面有headers属性请求头，通过请求头经常给服务器端携带公共参数，可以在network里看到
    //config.headers.token='123'
    // 这里的配置对象还需要进行返回，如果不进行返回可能到时候请求也发送不出去。
    return config
})

// 封装响应拦截器
// 响应拦截器会有两个回调，一个是成功的回调一个是失败的回调
request.interceptors.response.use((response) => {
    // 成功的话就将服务器里的数据进行返回
    // 简化数组
    // 因为成功的话会返回状态码和数据，在这里我们只需要返回数据即可
    return response.data
}, (error) => {
    // 失败的回调，一般处理Http的网络错误
    // 定义一个变量，用来存储网络错误的信息
    let message = ''
    // http状态码
    let status = error.resonse.status
    switch (status) {
        case 401:
            message: "token过期";
            break;
        case 403:
            message: "无权访问";
            break;
        case 404:
            message: "请求地址错误";
            break;
        case 500:
            message: "服务器出现问题";
            break;
        default:
            message:"网络出现问题";
            break;
    }
    // 提示错误信息 用了一个element组件
    ElMessage({
        type:'error',
        // kv一致省略v
        message
    })
    // 返回失败的Promise对象终结Promise链
    return Promise.reject(error)
})
// 对外暴露
export default request

组件中测试axios​

<template>
    <div>
        <h1>测试axios的二次封装</h1>
    </div>
</template>
<script setup lang="ts">
import request from '@/utils/request'
import { onMounted } from 'vue';
// 当组件挂载完毕测试发送一个请求
onMounted(() => {
    request({
        // 里面axios已经带有api了这里就不要再添加了
        url: '/user/login',
        method: 'post',
        data: {
            username: 'admin',
            password: "111111"
        }
    }).then(res => {
        // 相应成功了就用res来获取服务器返回数据
        console.log(res)
    })
})
</script>
