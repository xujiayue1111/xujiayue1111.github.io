mock

首先要安装插件​

这里 vite-plugin-mock 是需要依赖于 mockjs，所以两个依赖要一起安装

pnpm install -D vite-plugin-mock mockjs

在 vite.config.js 配置文件启用插件。​

import { UserConfigExport, ConfigEnv } from 'vite'
// 这里插件引入了viteMockServe函数，要在plugins之中调用viteMockServe函数
import { viteMockServe } from 'vite-plugin-mock'
import vue from '@vitejs/plugin-vue'
// 这里暴露了一个箭头函数 箭头函数返回配置对象。
// 如果这里暴露的是配置对象，就改成箭头函数吧。
export default ({ command })=> {
  return {
    plugins: [
      vue(),
      viteMockServe({
        // 这里的localEnabled要搭配vite-plugin-mock的2.9.6的版本不然会报错
        // 下载vite-plugin-mock的时候会默认下载3.0.0的版本，但是我这里使用enable关键字还是会报错
        // 所以还是用2.9.6版本搭配localEnable关键字吧
        // command用于获取当前开发的环境
        localEnabled: command === 'serve', //保证开发阶段可以使用mock接口
      }),
    ],
  }
}

在根目录创建mock文件夹：去创建我们需要mock数据与接口​

在mock文件夹内部创建一个user.ts文件，在这里mock文件用于创建假的接口

从 npm 官网里面找插件 vite-plugin-mock，找到之后看 code 文件夹就可以看到里面的接口到底是怎么书写的

安装axios​

pnpm install axios
import axios from 'axios'
// 登录接口测试 这里F12开发者工具点击network刷新得到login可以查看里面相应的消息
axios({
    url:"/api/user/login",
    method:"post",
    //这里带的是用户名和密码信息
    data:{
        username:"admin",
        password:"111111"
    }
})

最后通过axios测试接口能不能用。
